- 🎉 基于 Tauri v2 开发，轻量高效，跨平台体验更进一步。
- 💻 支持 Windows、macOS 和 Linux（x11），在多设备间无缝切换。
- ✨ 简洁直观的用户界面，操作简单，零门槛上手，开箱即用。
- 📋 支持纯文本、富文本、HTML、图片和文件类型的剪贴板内容。
- 🔒 数据本地存储，确保用户隐私安全，数据完全掌控在用户手中。
- 📝 支持备注功能，轻松分类、管理和检索，让工作更高效。
- ⚙️ 丰富的个性化设置，满足不同用户需求，打造专属体验。
- 🤝 完善的文档与社区支持，与开发者共同探索与成长。
- 🧩 持续优化中，更多惊喜功能等你发现。

## 🚀 Fork 版本增强功能

> 本仓库是 [EcoPasteHub/EcoPaste](https://github.com/EcoPasteHub/EcoPaste) 的 Fork 版本，在原版基础上增加了以下易用性改进：

### 📏 文本显示行数配置

- **功能说明**：可自定义剪贴板条目中纯文本、富文本、HTML 格式内容的显示行数
- **配置位置**：偏好设置 → 剪贴板 → 显示设置 → 文本显示行数
- **可选范围**：1 - 20 行（默认 4 行）
- **效果**：
  - 内容行数 ≤ 设定值时，按实际行数显示
  - 内容行数 > 设定值时，按设定行数截断显示

### 🖼️ 图片显示高度配置

- **功能说明**：可自定义剪贴板中图片预览的最大高度
- **配置位置**：偏好设置 → 剪贴板 → 显示设置 → 图片显示高度
- **可选范围**：50 - 500 像素（默认 100 像素）
- **效果**：图片预览高度超过设定值时，保持宽高比缩放显示

### 🔄 展开/收起功能

- **功能说明**：当内容超过显示限制时，提供展开/收起按钮，展开后可预览完整内容，收起后按设置限制截断显示
- **适用范围**：
  - 文本内容：超过设定行数时显示展开按钮
  - 图片内容：超过设定高度 **且** 缩放后未布满容器宽度时显示展开按钮
- **操作方式**：点击条目底部的展开/收起按钮切换显示状态
- **状态持久化**：展开/收起状态不受虚拟滚动影响，滚动后状态保持不变

### 📌 默认收起功能

- **功能说明**：可配置激活窗口时是否自动收起所有已展开的内容
- **配置位置**：偏好设置 → 剪贴板 → 显示设置 → 默认收起
- **行为**：
  - **开启**：每次激活窗口时，超出容器显示设置的内容自动收起
  - **关闭**（默认）：展开/收起状态始终保持，不受窗口激活影响

### 📍 窗口跟随光标位置（Windows）

- **功能说明**：剪贴板窗口可跟随输入光标位置显示
- **配置位置**：偏好设置 → 剪贴板 → 窗口设置 → 窗口位置 → 跟随光标
- **工作原理**：
  1. 优先获取当前输入框中的文本光标位置
  2. 如无法获取（如非文本输入场景），则回退到跟随鼠标位置
- **平台支持**：目前仅支持 Windows 系统
- **使用场景**：适合在文档编辑、代码编写等需要频繁粘贴的场景使用

### 🎯 不夺焦窗口体验（Windows）

- **功能说明**：呼出剪贴板窗口时，原应用保持焦点不变，体验与 Windows 自带 Win+V 剪贴板一致
- **工作原理**：
  - 按快捷键呼出窗口时，窗口显示后立即将焦点归还给原应用
  - 原应用中的输入光标不会消失，编辑状态完整保留
  - 点击剪贴板条目后直接粘贴到原应用，窗口自动隐藏
  - 需要搜索时，点击剪贴板窗口即可正常输入
  - 点击剪贴板窗口以外的区域自动隐藏窗口
- **使用流程**：
  1. 在任意应用中按快捷键 → 剪贴板窗口浮现，原应用焦点不变
  2. 鼠标点击或双击条目 → 内容粘贴到原应用，窗口自动消失
  3. 点击窗口外部 → 窗口自动隐藏
  4. 再次按快捷键 → 窗口隐藏
- **平台支持**：目前仅支持 Windows 系统

### ⚙️ 偏好设置分组优化

- 新增「显示设置」功能分组，合理归类显示相关配置项，包含：
  - 文本显示行数、图片显示高度、默认收起、操作按钮、自动排序、显示原内容
- 「内容设置」保留：自动粘贴、复制为纯文本、粘贴为纯文本、自动收藏、删除确认

### 🔧 配置持久化

- 所有新增配置项均自动保存到用户数据目录
- 应用更新后配置不会丢失
- 无需手动导出/导入配置

### 🔄 自动同步上游

- 本仓库每天自动检查上游 EcoPasteHub/EcoPaste 的更新
- 有新版本时自动合并并触发构建
- 如遇合并冲突会创建 Issue 通知手动处理

## 🐛 原版修复

> 以下是对原版 EcoPaste 已知问题的修复：

### 📋 剪贴板内容类型判断逻辑

- **问题描述**：
  - 复制网络图片时，应用可能错误识别为「HTML」类型而非「图片」类型
  - 截图后，应用将截图图片错误识别为「文件」类型而非「图片」类型
- **修复方案**：修改判断逻辑，当复制内容存在图片数据时，优先识别为「图片」类型；

### 💾 数据备份入口

- **问题描述**：数据备份功能入口缺失
- **修复方案**：显示数据备份功能入口，确保数据备份与恢复功能正常可用
